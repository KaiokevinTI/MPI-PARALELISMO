Relatório de Experimento: Processamento Paralelo de NLP com MPI1. Descrição do ProblemaO objetivo deste projeto é avaliar o desempenho e a escalabilidade do cálculo de similaridade de texto entre um grande volume de dados utilizando o algoritmo de Similaridade de Jaccard.Dataset: Quora Question Pairs (nlp_features_train.csv).Volume de Dados: Limite de 5.000 perguntas únicas, resultando em 12.497.500 comparações de pares.Tarefa: Tokenização, limpeza de texto e cálculo da interseção/união de palavras entre todos os pares possíveis.2. Metodologia de ParalelizaçãoPara otimizar o tempo de execução, o script serial original (avaliador.py) foi adaptado para um ambiente distribuído utilizando a biblioteca mpi4py (avaliadormpi.py).Divisão de Carga: O espaço de iteração do laço externo ($i$) é dividido entre os processos disponíveis.Comunicação: O processo mestre (Rank 0) carrega o dataset e realiza o broadcast das perguntas para todos os trabalhadores. Ao final, utiliza o gather para consolidar os resultados.Ambiente de Teste: Execução variando entre 1, 2, 4, 8 e 12 processos (threads).3. Resultados ObtidosOs testes demonstraram uma redução significativa no tempo de execução à medida que o número de processos aumentou, porém com rendimentos decrescentes em termos de eficiência.Gráficos de DesempenhoAs capturas de tela abaixo detalham as métricas coletadas:Tempo vs. Processos: O tempo caiu de aproximadamente 40s (1 proc) para cerca de 13s (12 procs).Speedup: O ganho de velocidade máximo atingiu cerca de 3.15x com 12 processos.Eficiência: A utilização efetiva do hardware caiu de 100% para aproximadamente 26% ao atingir 12 processos.4. Análise CríticaGargalo de Comunicação: A queda na eficiência (especialmente após 8 processos) sugere que o tempo gasto transferindo listas de resultados via MPI e o overhead de gerenciamento começaram a superar o ganho do processamento.Desbalanceamento de Carga: A estratégia de divisão por blocos de índices $i$ resulta em processos iniciais trabalhando mais do que os finais (já que o laço interno diminui de tamanho), o que limita o speedup ideal.Conclusão: O uso de MPI foi eficaz para reduzir o tempo total em mais de 65%, mas para escalas maiores, uma distribuição cíclica de tarefas seria necessária para manter a eficiência alta.
